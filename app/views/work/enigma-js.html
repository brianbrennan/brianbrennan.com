Enigma.js

Summer 2015 — Fall 2015

Enigma.js is an encryption engine designed after the WW2-era Enigma machine. Using an key-cipher encryption method, Enigma.js allows the user to safely encrypt information in their web application by using a secret key of their own choosing. Published using NPM, Enigma.js is ready to be dropped into a personal project of your choosing. Enigma.js is written in pure JavaScript, and is perfect for web based encryption, or Node applications.*

View On NPM View on Github

Users have the option to choose their encryption method, be it simple letters (base 52) or a much more robust semi-full ASCII table (base 92). By using semi-full ASCII, encrypted information is safer than war messages during WW2. To see a simple rundown of how this software was built in its earliest phases, as well as learning how it works in principle, please read Rebuilding the Enigma.

*Since Enigma.js is in the early stage of development, it is not designed to be used in production level applications.

How to Install

If you’re using NPM as a means of managing your node modules, than installing enigma.js is as simple as typing:

npm install enigma.js --save

Note that the --save suffix is an optional piece that will add enigma.js to your package.json dependencies

Instantiation

In order to initiate enigma.js in your Node Application, you must add it as a module to your main application javascript file. Below is an example of doing just that.

var Enigma = require('./enigma');

Once the module has been added to your application, an Enigma object must be added to your application in order to be used. Below is an example of instantiation 

var e = new Enigma({
	sequence: 'ascii',
	rotorSettings: 'ae3',
	spaces: true
});

Note the parameter object in the instantiation of our object. Enigma.js has three fields to be added upon instantiation in the parameter object. 

rotorSettings 	→ 	can be any string. This will act as your key 
					for they encryption

sequence 	→ 	can be ‘caps’,’ascii’, or ‘alphanumeric’. 
				This will change the output of the encryption 
				to the desired sequence.

spaces 		→ 	boolean. Determines if the output 
				will add in spaces or 
				not to the output.

Methods 

encrypt(String)

Will encrypt any string provided using the parameters set upon instantiation. Returns the encrypted message. 

decrypt(String)

Will decrypt any string provided that has already passed through the encrypt method. The settings on the object must be identical in order for proper decryption to occur.

That’s it! Using these two simple methods, Enigma can provide String encryption based on cipher-key algorithms. If you’d like to see what this encryption looks like in action, please visit the Enigmate page, or the Enigmate site itself. You can also view the code on NPM, or Github.

Below are examples of actual encryption/decryption inputs and outputs.

ASCII 

//Settings
//sequence → ascii
//spaces → false
//rotorSettings → hello

Welcome to the Jungle → <O[Whkh($$73,.KzMKISQ

Caps 

//sequence → caps
//rotorSettings → hello
//spaces → true

WELCOME TO THE JUNGLE → ANZVMPM LL ATV KAYWGE

NOTE: input using the caps sequence must be capital letters!

Alphanumeric

//sequence → alphanumeric
//rotorSettings → heelo
//spaces → true

Welcome to the Jungle → mprdkdQ VL H0s pvjYYM

NOTE: input using the caps sequence must be Alphanumeric characters!
